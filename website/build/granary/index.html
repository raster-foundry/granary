Granary

  • Getting started

A job runner for geospatial machine learning

Run models in AWS Batch without worrying about infrastructure more than once

Create a model with AWS resources configured once, then never worry about remembering the job definition or job queue names again for the rest of your life.

Keep track of your predictions and their results

Granary associates prediction runs with your models and keeps track of where their results are stored. With signed urls, Granary makes serving predictions stored on S3 to client applications easy.

Bring some sanity to your day-to-day modeling work in AWS Batch

Granary puts some guardrails in front of your interaction with Batch so you don't fall in to common mistakes, like mis-formatted arguments,missing arguments, and bad string values for important AWS resources.
